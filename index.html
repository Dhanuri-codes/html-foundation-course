Why Learn HTML First?

HTML (HyperText Markup Language) is the skeleton of every website on the internet. Before you can style a page with CSS or add interaction with JavaScript, you need to know HTML. It is the language that defines what content exists on a page and what role each piece plays.

This free HTML Foundation Course by Dhanuri Codes covers every core concept — from the very first tag you will ever write to semantic elements that help search engines understand your content. All lessons include real code examples, live browser previews, and quiz questions to make sure the knowledge sticks.

🌐

Used on Every Website

Every page on the internet — from Google to YouTube to simple blogs — is built on HTML. It is the universal language of the web.

🚀

Gateway to Web Dev

HTML is always the first step. Once you master it, learning CSS and JavaScript becomes far easier and more logical.

🔍

Improves Your SEO

Good HTML structure directly affects how search engines rank your pages. Semantic tags like <article> and <nav> tell Google what your content means.

What is HTML?
The language behind every webpage
5 min▶

HTML stands for HyperText Markup Language. It is not a programming language — there is no logic, no loops, no calculations. It is a markup language, which means you use tags to label and describe the content on a page so the browser knows how to display it.

Think of a web page like a newspaper. Someone has to decide: "This text is a headline. This is the main story. This is a photo. This is a caption." HTML is the tool that makes those decisions. The browser reads your HTML and translates those labels into what users see.

🨘
HTML
Structure — what content exists and what it means
🎨
CSS
Style — colours, fonts, layout, spacing
⚡
JavaScript
Behaviour — clicks, animations, dynamic data
🧠
Core rule: HTML = Structure. CSS = Style. JavaScript = Behaviour. You always write HTML first. Without it, CSS and JavaScript have nothing to work with.

🕒 A Brief History

HTML was invented by Tim Berners-Lee in 1991 while he worked at CERN, the physics research centre in Switzerland. He wanted scientists to share documents across different computers easily. The very first webpage is still live at info.cern.ch. Today we use HTML5, the fifth and current major version, which added powerful new elements for video, audio, forms, and semantic structure.

1991
HTML 1.0 — Tim Berners-Lee publishes the first version
1995
HTML 2.0 — first formal standard is published
1997
HTML 3.2 & 4.0 — tables, scripts, stylesheets introduced
2014
HTML5 — video, audio, canvas, semantic tags, mobile-ready
Now
HTML5 is the living standard — maintained and updated continuously
💡
Fun fact: The very first website had no images, no CSS, and no JavaScript — just plain HTML text with links. And it still works in your browser today.

Tags & Elements
The core building blocks of HTML
8 min▶

Everything in HTML is made of tags. A tag is a keyword written inside angle brackets. Most tags come in pairs: an opening tag that marks where something starts, and a closing tag (with a forward slash) that marks where it ends. Together they form an element.

<p>This is a paragraph.</p>
Opening tag Content Closing tag

🔗 Self-Closing Tags

Some tags insert content rather than wrapping it. They have no closing tag because there is nothing to wrap.

example.html
<!-- Adds a line break -->
<br>

<!-- Draws a horizontal line -->
<hr>

<!-- Inserts an image -->
<img src="photo.jpg" alt="A photo">

<!-- Links a stylesheet -->
<link rel="stylesheet" href="style.css">

📋 Attributes

Attributes are extra details you add inside an opening tag. They follow the format name="value" and always go after the tag name, separated by a space. You can have multiple attributes on one tag.

<!-- One attribute -->
<p id="intro">Welcome!</p>

<!-- Multiple attributes -->
<a href="https://example.com" target="_blank" title="Visit Example">

</a>

Page Structure
The boilerplate every HTML file starts with
7 min▶

Every HTML page — whether it is a simple blog post or a complex web app — follows the same basic skeleton. You write this structure once at the start of every file. Editors like VS Code can generate it automatically by typing ! and pressing Tab.

index.html — the full boilerplate
<!-- 1. Declare HTML5 -->
<!DOCTYPE html>

<!-- 2. Root element; lang helps screen readers & SEO -->
<html lang="en">

<head>
  <!-- 3. Character encoding — always UTF-8 -->
  <meta charset="UTF-8">
  <!-- 4. Makes the page mobile-responsive -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <!-- 5. Page title — shows in browser tab & search results -->
  <title>My Awesome Page</title>
  <!-- 6. SEO description -->
  <meta name="description" content="A short page description for Google.">
  <!-- 7. Link to CSS stylesheet -->
  <link rel="stylesheet" href="style.css">
</head>

<body>
  <!-- All visible content goes here -->
  <h1>Hello, World!</h1>
  <p>This is my first webpage.</p>
</body>

</html>
<!DOCTYPE html>

Tells the browser to use HTML5. Without it, the browser enters "quirks mode" and renders incorrectly.

<html lang="en">

The root wrapper. lang="en" helps Google and screen readers know the page language.

<head>

Contains invisible page settings: title, charset, meta tags, links to CSS/JS. Nothing here shows on screen.

<meta charset="UTF-8">

Ensures all characters (including ©, é, 中) display correctly. Always include this first inside <head>.

<meta name="viewport">

Makes your page scale properly on phones and tablets. Essential for mobile-responsive design.

<title>

The text shown on the browser tab and in Google search results. Keep it descriptive and under 60 characters.

<body>

Everything the user actually sees and interacts with goes here — headings, text, images, buttons, forms.

</html>

Closes the root element. This must be the very last line of every HTML file.

🧠
Golden rule: Anything in <head> is invisible to the user. Anything in <body> is visible. Never mix them up.

Headings & Paragraphs
Structuring text content the right way
8 min▶

Text is the backbone of most web pages. HTML gives you two primary tools for organising it: headings (which define hierarchy and importance) and paragraphs (which hold your body text). Used correctly, they also help search engines understand and rank your content.

📈 Heading Hierarchy — h1 to h6

There are six heading levels. Use them in order — never skip a level. An h1 is the main title of the page, h2 is a major section, h3 is a subsection of that, and so on. Think of it like a document outline.

headings.html
<h1>Main Page Title</h1>
<h2>Chapter 1: Getting Started</h2>
<h3>Section 1.1: Installation</h3>
<h4>Step A: Download the file</h4>
<h5>Tip for Windows users</h5>
<h6>Advanced note</h6>

📝 Paragraphs, Bold, Italic & More

text-formatting.html
<p>A standard paragraph of text goes here.</p>

<!-- Bold (important) -->
<p>This word is <strong>very important</strong>.</p>

<!-- Italic (emphasis) -->
<p>This is <em>emphasised</em> text.</p>

<!-- Underline -->
<p>This is <u>underlined</u>.</p>

<!-- Inline code -->
<p>Use the <code><br></code> tag for line breaks.</p>

<!-- Line break inside a paragraph -->
<p>Line one.<br>Line two — same paragraph.</p>
🧠
SEO tip: Every page should have exactly one <h1>. It tells search engines what the whole page is about. Using multiple h1 tags confuses Google and hurts your ranking.

5
Links & Images
Connecting pages and displaying visuals
9 min▶

Links are what make the web a "web" — they connect pages together. Images make pages visually engaging. Both use straightforward tags with a few important attributes to master.

🔗 Anchor Tag — <a>

links.html
<!-- Internal link (same website) -->
<a href="about.html">About Us</a>

<!-- External link (another site) — opens in new tab -->
<a href="https://github.com" target="_blank" rel="noopener noreferrer">GitHub</a>

<!-- Email link -->
<a href="mailto:hello@example.com">Email Us</a>

<!-- Jump to a section on the same page -->
<a href="#contact">Go to Contact Section</a>

<!-- Phone number link (mobile) -->
<a href="tel:+94777123456">Call Us</a>
💡
Security tip: When using target="_blank", always add rel="noopener noreferrer". This prevents the new tab from accessing your page and is a web security best practice.

📷 Image Tag — <img>

images.html
<!-- Local image file -->
<img src="images/banner.jpg" alt="A banner showing a mountain sunset" width="800" height="400">

<!-- Image from the internet -->
<img src="https://example.com/photo.jpg" alt="Description of the photo">

<!-- Clickable image (image inside a link) -->
<a href="https://example.com">
  <img src="logo.png" alt="Company Logo">
</a>
🧠
Always include alt text! The alt attribute describes an image to screen readers (for blind users) and also appears if the image fails to load. Google also reads alt text to understand your images — it improves SEO.

Bullet points, numbered steps, and descriptions

Lists are one of the most used HTML structures. Navigation menus, feature lists, step-by-step guides, ingredients, and site maps — they all use HTML lists under the hood.

HTML Tables
Displaying data in rows and columns
9 min▶

HTML tables are used to display data that belongs in a grid — price lists, schedules, comparison charts, results tables, and spreadsheet-style information. Tables are made of rows, and rows are made of cells.

📈 Basic Table Structure

table.html
<table>
  <!-- Table head -->
  <thead>
    <tr>
      <th>Language</th>
      <th>Used For</th>
      <th>Difficulty</th>
    </tr>
  </thead>
  <!-- Table body -->
  <tbody>
    <tr>
      <td>HTML</td>
      <td>Web structure</td>
      <td>Easy</td>
    </tr>
    <tr>
      <td>CSS</td>
      <td>Web styling</td>
      <td>Medium</td>
    </tr>
  </tbody>
</table>

📋 Merging Cells — colspan & rowspan

<!-- colspan: cell spans 2 columns -->
<td colspan="2">Merged cell</td>

<td rowspan="3">Tall cell</td>
🧠
Important: Use tables only for tabular data (grids of information). Do not use tables to create page layout — that is what CSS Flexbox and Grid are for.

HTML Forms
Collecting input from users
10 min▶

Forms are how users interact with websites — login pages, search bars, contact forms, sign-up sheets, and checkouts are all built with HTML forms. Every form sends data somewhere when submitted.

📝 Basic Form Structure

form.html
<form action="/submit" method="post">

  <!-- Text input -->
  <label for="name">Full Name</label>
  <input type="text" id="name" name="name" placeholder="Enter your name" required>

  <!-- Email input -->
  <label for="email">Email</label>
  <input type="email" id="email" name="email" required>

  <!-- Dropdown -->
  <label for="level">Your Level</label>
  <select id="level" name="level">
    <option value="beginner">Beginner</option>
    <option value="advanced">Advanced</option>
  </select>

  <!-- Textarea -->
  <label for="msg">Message</label>
  <textarea id="msg" name="msg" rows="4"></textarea>

  <!-- Checkbox -->
  <input type="checkbox" id="agree" name="agree">
  <label for="agree">I agree to the terms</label>

  <!-- Submit button -->
  <button type="submit">Send Message</button>

</form>

Semantic HTML
Writing HTML that means something
10 min▶

Semantic HTML is one of the most important concepts in modern web development. It means using HTML tags that describe the meaning of the content they wrap — not just how it should look. Instead of using a generic <div> for everything, semantic HTML tells the browser and search engines exactly what role each block plays.

🔃 div vs Semantic Tags

✘ Non-semantic (confusing)
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>
<div class="sidebar">...</div>
<div class="footer">...</div>
✔ Semantic (clear meaning)
<header>...</header>
<nav>...</nav>
<main>...</main>
<aside>...</aside>
<footer>...</footer>

📋 Key Semantic Elements

TagMeaningUse it for
<header>Page or section headerLogo, site name, top navigation
<nav>NavigationMain menu, breadcrumbs, pagination
<main>Main contentThe primary content of the page — only one per page
<article>Self-contained contentBlog posts, news articles, forum posts
<section>Thematic groupChapters, tabs, named areas within a page
<aside>Sidebar contentRelated links, ads, author bio
<footer>Page or section footerCopyright, contact links, social icons
<figure>Media with captionImage + caption, diagram, code with explanation
<figcaption>Caption for a figureText beneath a figure element
<time>Date or timePublished dates, event times
<mark>Highlighted textSearch results, key terms
<address>Contact informationEmail, phone, physical address in a footer
🧠
Why it matters for SEO: Google uses semantic tags to understand the structure and importance of content. A page with proper semantic HTML ranks higher than the same page built with only <div> tags, because Google can identify your headings, navigation, main article, and footer automatically.

HTML Best Practices
Writing clean, accessible, and SEO-ready code
8 min▶

Knowing the tags is only half the job. Writing HTML that is clean, accessible, maintainable, and optimised for search engines is what separates beginners from professional developers. Here are the rules that matter most.

✅ The Golden Rules

01
Always close your tags

Forgetting to close a tag causes the browser to guess where it ends. This breaks layout in unpredictable ways.

<!-- Wrong -->
<p>This paragraph never ends

<!-- Correct -->
<p>This paragraph is properly closed.</p>
02
Use lowercase for all tags and attributes

HTML5 is case-insensitive but industry convention is always lowercase. Mixed case looks unprofessional and causes bugs in some tools.

<!-- Wrong -->
<IMG SRC="photo.jpg">

<!-- Correct -->
<img src="photo.jpg" alt="Description">
03
Always quote attribute values

While some attributes work without quotes in HTML5, always use them. It prevents ambiguity and is required in XML-based documents.

<!-- Risky -->
<p class=intro>

<!-- Safe -->
<p class="intro">
04
Indent nested elements

Proper indentation (2 or 4 spaces) makes code readable. Anyone maintaining your code — including future you — will thank you.

05
Write descriptive alt text for every image

Not just "image" or "photo" — describe what is actually in the image. This serves blind users and helps Google index your images.

06
Use semantic tags, not only divs

Replace <div class="header"> with <header>. Replace <div class="nav"> with <nav>. This improves both SEO and accessibility.

07
Always add the viewport meta tag

Without it, your page will look zoomed out on mobile devices. Google also uses mobile-friendliness as a ranking signal.

08
One <h1> per page, used meaningfully

Your h1 should contain the primary keyword for the page. It is the single most important piece of on-page SEO in your HTML.

Reference Guide

📋 HTML Cheat Sheet

Every tag covered in this course, in one place. Bookmark this page and come back whenever you need a quick reference.

Document Structure
<!DOCTYPE html>
Declares the document as HTML5. Always the very first line.
<html lang="en">
Root wrapper. lang attribute aids screen readers and SEO.
<head>
Container for invisible settings: title, charset, CSS links.
<body>
Everything visible to the user lives here.
<title>
Page title shown in the browser tab and Google results.
<meta charset="UTF-8">
Ensures all characters display correctly. Always first in <head>.
Text & Headings
<h1> – <h6>
Headings. h1 is the most important. Use only one h1 per page.
<p>
A paragraph of text. Gets automatic spacing above and below.
<strong>
Bold + important. Signals importance to browsers and Google.
<em>
Italic + emphasis. Signals stressed emphasis to screen readers.
<br>
Line break inside a paragraph. Self-closing — no closing tag.
<hr>
Horizontal dividing line. Thematic break between sections.
<code>
Inline code — monospace font, used for variable names or commands.
<blockquote>
An extended quotation from another source, indented by default.
Links & Media
<a href="...">
Anchor tag — creates a clickable hyperlink.
<img src="..." alt="...">
Embeds an image. Always include descriptive alt text.
<video>
Embeds a video file with native browser controls.
<audio>
Embeds an audio file with native browser controls.
<figure>
Groups media (image, video) with its caption.
<figcaption>
The caption text for a <figure> element.
Lists
<ul>
Unordered list — bullet points. Order doesn't matter.
<ol>
Ordered list — numbered. Order matters (steps, rankings).
<li>
A single list item inside <ul> or <ol>.
<dl> <dt> <dd>
Description list — term + definition pairs. Used for glossaries.
Semantic Layout
<header>
Page or section header — logo, site name, top nav.
<nav>
Navigation links — main menu, breadcrumbs.
<main>
Primary content. Use only once per page.
<article>
Self-contained content — blog post, news item, forum post.
<section>
Thematic grouping within a page — chapters, named areas.
<aside>
Sidebar content — related links, ads, author bio.
<footer>
Page or section footer — copyright, links, contact.
<div>
Generic block container — use when no semantic tag fits.
<span>
Generic inline container — for styling a word or phrase.
Forms
<form>
Wraps all form elements. Has action and method attributes.
<input>
Interactive input field. type attribute sets what kind.
<label>
Clickable label for an input. Always link with for="id".
<textarea>
Multi-line text input area for long messages.
<select> <option>
Dropdown menu — <select> wraps <option> items.
<button>
Clickable button. type="submit" sends the form.
